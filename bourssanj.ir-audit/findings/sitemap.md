# Sitemap Audit — bourssanj.ir

Source: `https://bourssanj.ir/sitemap.xml` (referenced correctly from `robots.txt`), generated by `app/sitemap.ts`.

## Summary Stats
- URLs found: 1630 (`<url>` count) — well under the 50,000-per-file Google limit, no sitemap index needed.
- XML validity: valid (parsed cleanly with `xml.etree.ElementTree`).
- Sample of 15 URLs spread across the file (every ~110th entry, including `/`, `/fund/*`, `/stock/*`, `/technical/*`) all returned HTTP 200. No 404s or redirects detected in sample.
- Breakdown by top-level path: `/technical` 665, `/stock` 662, `/fund` 273, `/trade-value` 10, `/monitor` 6, `/funds` 3, `/analysis` 3, `/valuation` 2, plus 1 each for `/`, `/stocks`, `/vip`, `/track-record`, `/signals`, `/compare`.

## Findings

### 1. `/market-map` page missing from sitemap
- **Severity:** High
- **Description:** `app/market-map/` is a real, publicly crawlable route (verified `200 OK`, not disallowed in `robots.txt`), but it is absent from `STATIC_ROUTES` in `app/sitemap.ts`, so it never appears in the generated sitemap.
- **Recommendation:** Add `{ path: '/market-map', priority: 0.7 }` (or appropriate priority) to `STATIC_ROUTES` in `app/sitemap.ts`.

### 2. `/alerts` page missing from sitemap
- **Severity:** Medium
- **Description:** `app/alerts/page.tsx` returns `200 OK` and is not blocked by `robots.txt`, yet it is not listed in `STATIC_ROUTES`.
- **Recommendation:** Add `/alerts` to `STATIC_ROUTES` if it is meant to be a public, indexable feature page. If it is actually gated behind login (verify manually — the anonymous response was 200), instead disallow it in `robots.txt` and drop it from indexing entirely for clarity.

### 3. `/fundamentals/[symbol]` pages entirely absent from sitemap
- **Severity:** Medium
- **Description:** Unlike `/stock/[symbol]` and `/technical/[symbol]`, the dynamic route `app/fundamentals/[symbol]/page.tsx` has no corresponding entry generator in `sitemap.ts`. Spot check confirms `/fundamentals/وبملت` returns `200 OK`, meaning ~660+ real indexable pages (one per stock symbol) are undiscoverable via sitemap.
- **Recommendation:** In `app/sitemap.ts`, reuse the `symbols` list (already fetched for `stockEntries`/`technicalEntries`) to also emit `fundamentalsEntries` mapped to `${SITE_URL}/fundamentals/${symbol}`.

### 4. `lastmod` is always "now", not real content dates
- **Severity:** Low
- **Description:** `sitemap()` sets `const lastModified = new Date()` once per request and reuses it for every entry (confirmed: every URL in the fetched file shares timestamp `2026-07-18T08:48:57.129Z`). This tells Google nothing about actual freshness and can cause crawlers to under-trust the signal or ignore it.
- **Recommendation:** For static marketing/tool pages, use each page's real deploy/content-change date (or a coarse per-section date) instead of request time. For `/fund/[slug]`, `/stock/[symbol]`, `/technical/[symbol]`, prefer the underlying data's last-updated timestamp (e.g., last snapshot/report date already tracked in Supabase) rather than the moment the sitemap was rendered.

### 5. `priority` and `changeFrequency` fields present (Google ignores both)
- **Severity:** Info
- **Description:** `sitemap.ts` sets `priority` (0.4–1) and `changeFrequency: 'hourly'/'daily'` on every entry. Google has publicly stated both fields are ignored.
- **Recommendation:** Optional cleanup — safe to remove to slightly shrink file size and simplify the generator; no ranking impact either way.

### 6. `/vip` base page returns 404, only `/vip/filters` is real
- **Severity:** Info
- **Description:** `/vip` itself 404s; only `/vip/filters` exists and is correctly the only VIP entry in the sitemap. No action needed — flagged only to confirm this isn't an accidental gap.

## Location Page / Doorway Page Quality Gate
Not applicable in the traditional "city-swapped landing page" sense — this site has no location-based pages. The programmatic `/stock/[symbol]`, `/fund/[slug]`, `/technical/[symbol]` pages (1600+ combined) are analogous to the "Product pages (unique specs)" safe pattern: each carries genuinely unique, per-symbol financial data (prices, technical indicators, fund NAV/holdings) rather than templated text with only a name swapped, so no doorway-page penalty risk was identified. No warning/hard-stop triggered.
